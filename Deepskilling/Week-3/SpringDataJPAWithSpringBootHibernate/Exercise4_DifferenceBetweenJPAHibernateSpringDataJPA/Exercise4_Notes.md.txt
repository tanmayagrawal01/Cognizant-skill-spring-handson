# Difference between JPA, Hibernate and Spring Data JPA

## JPA (Java Persistence API)
- JPA is a specification (JSR 338).
- It defines how Java objects are mapped to database tables.
- It does not provide an implementation.

## Hibernate
- Hibernate is an ORM framework.
- It is one of the implementations of JPA.
- It converts Java objects into database records.

## Spring Data JPA
- Spring Data JPA is built on top of JPA.
- It reduces boilerplate code.
- It provides JpaRepository.
- It manages transactions using @Transactional.

## Comparison

| Feature | JPA | Hibernate | Spring Data JPA |
|---------|-----|-----------|-----------------|
| Type | Specification | ORM Framework | Spring Module |
| Implementation | No | Yes | Uses Hibernate |
| Boilerplate | High | Medium | Low |
| Transactions | Manual | Manual | Automatic |

## Hibernate Example

```java
Session session = factory.openSession();
Transaction tx = session.beginTransaction();
session.save(employee);
tx.commit();
session.close();
```

## Spring Data JPA Example

```java
@Repository
public interface EmployeeRepository extends JpaRepository<Employee, Integer> {
}

@Service
public class EmployeeService {

    @Autowired
    private EmployeeRepository employeeRepository;

    @Transactional
    public void addEmployee(Employee employee){
        employeeRepository.save(employee);
    }
}
```
